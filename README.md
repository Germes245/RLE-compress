# RLE Compress

Библиотека на C для сжатия и распаковки массивов байт (uint8_t) алгоритмом **RLE** (Run‑Length Encoding).  
Проект предназначен для простого и эффективного кодирования длинных серий одинаковых значений.

## Особенности

- Сжатие выполняется в формате пар: `[count, value]`, где `count` – число повторений (1…255), `value` – сам байт.
- Длинные серии (более 255 одинаковых байт) автоматически разбиваются на несколько блоков по 255.
- Функции используют динамическое выделение памяти (`malloc`) — вызывающий обязан освободить буфер через `free()`.
- Работает с любыми бинарными данными (интерпретирует их как `uint8_t`).
- Корректно обрабатывает ошибки (неверные указатели, нулевая длина, повреждённые сжатые данные).

## Сборка

Проект собирается с помощью **CMake** (требуется версия 3.10 или выше).

```bash
git clone https://github.com/Germes245/RLE-compress.git
cd RLE-compress
mkdir build && cd build
cmake ..
make
```

После сборки вы получите:

- динамическую библиотеку librle_lib.so (или .dylib / .dll в зависимости от платформы);
- тестовый исполняемый файл test_rle.

## Использование

Включите заголовочный файл и линкуйте библиотеку:

```c
#include "rle_compress_and_decompress_1_byte.h"

// Сжатие
size_t compressed_len;
char* compressed = rle_compress(original_data, original_len, &compressed_len);
if (compressed == NULL) {
    // обработка ошибки
}

// Распаковка
size_t decompressed_len;
char* decompressed = rle_decompress(compressed, compressed_len, &decompressed_len);
if (decompressed == NULL) {
    // обработка ошибки
}

// Не забудьте освободить память
free(compressed);
free(decompressed);
```

### Пример

Исходный массив: [255, 255, 255, 255, 255, 90, 90, 90, 90]<br/>
После сжатия: [5, 255, 4, 90] (4 байта вместо 9).
